🎬🎵 RecMe — AI-Powered Music & Movie Recommendation App
Full MVP Build Prompt
---
🧭 Overview
Build RecMe, an AI-powered recommendation web app that suggests music and movies based on a user's Spotify listening history, mood input, genre preferences, and trending data. The app should feel like a premium, editorial-grade product — not a generic dashboard.
App Name: RecMe  
Tagline: "Your taste. Amplified."  
Stack: Next.js 14 (App Router) + Supabase + Tailwind CSS + Vercel  
Auth: Spotify OAuth (for music) + Email/Google (for movie-only users)  
AI Engine: Anthropic Claude API (`claude-sonnet-4-20250514`)  
Movie Data: TMDB API (free)  
Music Data: Spotify Web API

🔁 Auth State Detection Logic (Next.js App Router)
```typescript
// app/page.tsx
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'
import GuestLanding from '@/components/landing/GuestLanding'
import PersonalizedHome from '@/components/landing/PersonalizedHome'

export default async function RootPage() {
  const cookieStore = cookies()
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { get: (name) => cookieStore.get(name)?.value } }
  )
  const { data: { session } } = await supabase.auth.getSession()

  if (!session) {
    // Fetch trending data server-side for guests
    const [trendingMovies, trendingMusic] = await Promise.all([
      fetchTMDBTrending(),     // server-side TMDB call
      fetchAICuratedTrending() // Claude-generated trending list (cached)
    ])
    return <GuestLanding trendingMovies={trendingMovies} trendingMusic={trendingMusic} />
  }

  // Fetch user data + personalized recs server-side
  const user = await getUserFromSupabase(session.user.id)
  const spotifyData = await getSpotifyTopData(user.spotify_access_token)

  return <PersonalizedHome user={user} spotifyData={spotifyData} />
}
```

🗄️ Supabase Schema
```sql
-- Users
create table users (
  id uuid primary key default gen_random_uuid(),
  email text unique,
  spotify_id text unique,
  spotify_access_token text,
  spotify_refresh_token text,
  display_name text,
  avatar_url text,
  preferences jsonb default '{}',  -- { music_genres: [], movie_genres: [], mood: "" }
  created_at timestamptz default now()
);

-- Chat Sessions
create table chat_sessions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id),
  type text check (type in ('music', 'movie')),
  messages jsonb default '[]',  -- [{role, content, timestamp}]
  created_at timestamptz default now()
);

-- Saved Recommendations
create table recommendations (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id),
  type text check (type in ('music', 'movie')),
  item_data jsonb,  -- full track or movie object
  saved_at timestamptz default now()
);
```

v0.3 — 2026-03-24 · Landing Page Smart Rendering
Change: The landing page (`/`) now conditionally renders two entirely different experiences based on auth state — `<GuestLanding />` for unauthenticated users and `<PersonalizedHome />` for logged-in users.
What changed vs v0.2:
Area	Before	After
Landing page	Static hero + login CTAs only	Smart SSR: guest sees trending, user sees AI recs
Auth detection	Client-side redirect after hydration	Server-side session check via `@supabase/ssr` (zero layout shift)
Guest experience	Just a login wall	Full trending content (TMDB + AI music), sticky CTA banner
Logged-in experience	Redirect to `/home`	Personalized hero, auto-triggered AI recs, inline chat
Route structure	`/` = landing, `/home` = dashboard	`/` = smart route (both states), `/home` = alias
Background	Static gradient	Color extracted from user's top album art (ColorThief)
AI recs trigger	Only on user action (chat input)	Auto-triggered on page load using Spotify + Supabase data
New components added:
`<GuestLanding />` — guest hero + trending + sticky CTA
`<PersonalizedHome />` — personalized hero + AI recs grid + inline chat
`<AIRecsGrid />` — magazine-layout asymmetric card grid
`<ColorThiefBackground />` — ambient color extraction from album art
`<StickySignInBanner />` — scroll-triggered sticky CTA for guests
`<TimeAwareGreeting />` — time-of-day-aware greeting with username
New API routes added:
`GET /api/tmdb/trending` — server-side TMDB fetch for guest landing (ISR, revalidate 3600s)
`GET /api/claude/trending-music` — Claude-generated music trending list for guest view (cached)
`GET /api/recs/auto` — auto-trigger personalized Claude recs on page load for logged-in users
Optimization decisions:
Guest trending data uses ISR (Incremental Static Regeneration) with 1-hour revalidation — avoids hitting TMDB on every guest visit
Personalized AI recs use React Suspense + streaming so the page shell renders instantly
Color extraction from album art runs client-side only (ColorThief.js) to avoid server overhead
Spotify token refresh is handled server-side before rendering to prevent stale token errors mid-page

v0.2 — 2026-03-24 · Core Architecture & Feature Design
Change: Full technical architecture, Supabase schema, API integrations, Claude system prompt, and all core screens defined.
Decisions made:
Next.js 14 App Router chosen for native streaming support and server components
Supabase chosen for Postgres-native JSON storage, better free tier, and row-level security
Claude claude-sonnet-4-20250514 chosen — best balance of speed and quality for real-time rec streaming
TMDB chosen over IMDB for free API access with no rate-limit issues at MVP scale
framer-motion for animations — more control than CSS-only for card reveals and tab transitions
SWR for client-side data fetching — built-in caching, deduplication, revalidation
Architecture decisions:
Single Supabase `recommendations` table stores both music and movies (by `type` column) — simpler schema for MVP
Spotify tokens stored in Supabase users table (encrypted at rest by Supabase)
Claude system prompt injects Spotify data at request time — keeps AI context fresh without fine-tuning
